Project Overview

This demo emulates a lightweight smart-parking solution where edge classification (two_wheeler / empty) runs on an ESP32-CAM and the results are shown in a polished, minimal dashboard. The demo intentionally avoids backend services — instead it shows how an embedded device’s output could be reflected in a production-ready UI.

Core components

  • Edge model: TinyML classifier (two_wheeler vs empty)
  • Device: ESP32-CAM (simulated in the UI)
  • Dashboard: HTML/CSS/JS with local state persistence

Next steps (optional)

  1. Wire real device via WebSocket / lightweight HTTP + JSON.
  2. Add basic authentication backed by a small server for real deployments.
  3. Store historical metrics and analytics for usage reports.
